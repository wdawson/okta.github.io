What if you‚Äôre a front-end developer? The React Sample Applications profile is a good starting point.

Sample Description
Okta-Hosted Login A React application that will redirect the user to the Okta-hosted login page of your org for authentication. The user is redirected back to the React application after authenticating.

In addition to showing you both of these applications, they can be integrated with a resource server too. Node/Express and Java/Spring examples are provided.

You can set up the React + Okta-hosted login sample quite easily if you already have Git installed:

git clone git@github.com:okta/samples-js-react.git
cd samples-js-react/okta-hosted-login
npm install

Then modify src/.samples.config.js to have your domain and client ID in it. Run it with npm start, and you‚Äôll see screenshots like the ones below.

To set up the Spring Boot resource server, clone that sample‚Äôs repository and run it with your issuer:

git clone https://github.com/okta/samples-java-spring.git
cd samples-java-spring/resource-server
./mvnw -Dokta.oauth2.issuer=https://{yourOktaDomain}.com/oauth2/default

The Express resource server requires little configuration to configure it too.

git clone git@github.com:okta/samples-nodejs-express-4.git
cd samples-nodejs-express-4/
npm install

Modify .samples.config.json for your app, then run npm run resource-server. You should see the same result as you did with the Java resource server.
